06

데이터
거버넌스

1~5단계를 통해 구축된 데이터 구조가 시간이 흐름에 따라 파편화되거나 오염되지 않도록, 전사적인 차원의 지속 가능한 관리 체계를 수립한다. 아키텍처를 단순한 설계도로 남겨두지 않고 기업의 지식 자산으로 승화시키는 최종 단계다.

① 전사 데이터 표준화 체계

전사적으로 데이터의 명칭·정의·형식·규칙을 통일하여 의사소통의 혼란을 방지하고 시스템 간 정합성을 확보한다. 카드를 클릭하면 실제 예시가 펼쳐진다.

📝

표준 단어 / 용어

비즈니스에서 사용하는 단어를 조합하여 표준 용어를 정의하고 용어 사전에 등록한다.

혼용 → 통일

사다구매주문 → 매수 로 통일

결제청산인도 → 각각 정확한 의미로 분리 정의
🏷️

표준 도메인

속성이 가질 수 있는 값의 범위, 데이터 타입, 길이를 표준화한다.

도메인 정의 예시

금액 도메인 → DECIMAL(18,0)
종목코드 → CHAR(12)
날짜 → DATE

→ 단수 차이 오차 원천 차단, 시스템 간 정합성 자동 확보
🔤

표준 명명 규칙

엔터티(테이블)와 속성(칼럼)의 이름을 짓는 일정한 규칙을 수립한다.

명명 패턴

테이블: 목적어_용도
ASSET_MSTTRADE_LOGACCOUNT_BAL

칼럼: 속성명_단위
PRICE_KRWQTY_SHR
📋

용어 사전 관리

정의된 모든 용어·도메인·규칙을 중앙에서 관리하는 저장소를 구축한다.

전사 효과

개발자 착각 방지 → 개발 오류 감소
신규 입사자 온보딩 가속화
시스템 연계 시 매핑 테이블 불필요

→ 전사 표준 준수율 99% 목표

② 메타데이터 저장소 — 4단계 순환

거버넌스의 기술적 기반은 메타데이터(데이터에 대한 데이터)와 이를 저장하는 중앙 저장소다. 아래 사이클이 지속적으로 돌아야 아키텍처가 살아있다. 노드를 클릭하면 세부 내용이 나타난다.

노드를 클릭하면 각 단계의 역할이 여기에 표시됩니다.

순환 구조: 수집 → 추적 → 분석 → 업데이트 → 수집 ...

③ 데이터 품질 6원칙

데이터 품질은 단일 지표로 측정할 수 없다. 6개 차원이 모두 높은 수준을 유지해야 진정한 데이터 품질이 확보된다. 스크롤하면 실측치가 채워진다.

④ 데이터 소유권 정의

설계된 모델이 실제 운영 환경에서도 품질을 유지하고 있는지 주기적으로 검증하고, 데이터의 생성·수정·폐기 책임을 명확히 한다. 행을 클릭하면 상세 정보가 펼쳐진다.

데이터 엔터티 소유 부서 권한 품질 기준

⑥ 아키텍처의 완성

🏆 이 6단계의 본질: 전략의 영속화

1단계에서 정의한 비즈니스 목표와 4~5단계에서 구현한 정교한 데이터 모델은 거버넌스라는 울타리가 없다면 시간이 지나면서 다시 엉키게 된다. 표준화 → 메타 관리 → 품질 검증 → 조직적 대응으로 이어지는 이 순환 고리를 시스템화함으로써, 기업이 어떠한 비즈니스 환경 변화에도 흔들리지 않는 데이터 중심 경영(Data-Driven Management)의 토대가 완성된다.
