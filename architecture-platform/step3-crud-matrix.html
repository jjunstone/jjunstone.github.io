CRUD 매트릭스

2단계에서 분할된 4개 주제 영역의 엔터티들이 어떤 업무 기능에 의해 생성(C)·조회(R)·수정(U)·삭제(D)되는지 매핑한다. 모든 데이터의 생성 소유권을 검증하고 기능–데이터 간 논리적 완결성을 증명한다.

CRUD 분석의 본질

아키텍처적으로 가장 중요한 원칙: 모든 엔터티는 반드시 하나 이상의 기능에 의해 생성(C)되어야 한다. 생성 주체가 없는 엔터티는 설계 오류를 의미한다. 카드를 클릭하면 해당 연산만 매트릭스에서 강조된다.

C — Create

생성

데이터 존재를 시작하는 행위. 소유권을 결정하는 핵심 연산.

R — Read

조회

다른 기능의 의사결정을 위해 읽는 행위. SA 간 의존성의 핵심.

U — Update

수정

상태를 갱신하는 행위. 이력 모델링(선분 이력)과 직결됨.

D — Delete

삭제(논리)

생명주기 종료. 금융 시스템에서는 물리 삭제 대신 논리 삭제·아카이브가 원칙.

7 × 12 CRUD 매트릭스

기능명(행)을 클릭하면 상세 설명이 펼쳐진다. 열 헤더를 클릭하면 해당 엔터티 열이 강조된다.

필터:
1

핵심 검증: 생성 소유권 (12 / 12)

전체 12개 엔터티 모두 정확히 하나 이상의 기능에 의해 C(Create)되어야 한다. 생성 주체가 없는 '고아 엔터티(Orphan Entity)'는 설계 누락을 의미한다. 점을 클릭하면 생성 주체가 아래에 표시된다.

노드를 클릭하면 해당 엔터티의 생성 주체가 여기에 표시됩니다.

색상: 🔵 자산 관리 · 🔴 거래 관리 · 🟢 위험 관리 · 🟡 원장 관리

데이터 생성 순서도

C(Create) 연산을 업무 흐름 순서로 정렬하면 데이터가 어떤 순서로 탄생하는지 드러난다. 노드를 클릭하면 해당 단계의 설명이 펼쳐진다.

SA별 소유권 클러스터링

C(Create) 연산을 대각선 방향으로 정렬(Clustering)하면 독립적인 업무 모듈이 자연스럽게 분리된다. 아래는 실제 매트릭스의 C셀만 추출한 패턴이다 — 칸에 마우스를 올리면 어떤 기능×엔터티인지 보인다.

왼쪽 위(① 마스터 관리)에서 오른쪽 아래(⑦ 원장 마감)로 이어지는 계단형 패턴이 보인다. 이것이 업무 프로세스 순서와 데이터 소유권이 일치한다는 시각적 증거다.

📦 자산 관리 SA

ASSET ASSET_CLASS MARKET_DATA
① 상품/한도 마스터 관리 → ASSET, ASSET_CLASS
② 시장 시세 수신 → MARKET_DATA
타 SA는 R(참조)만 수행 → Low Coupling 유지

🔄 거래 관리 SA

ORDER EXECUTION SETTLEMENT
③ 매매 주문 접수 → ORDER
④ 주문 체결 처리 → EXECUTION
⑤ 정산 처리 → SETTLEMENT
주문→체결→정산 거래 생명주기 순서대로 생성

⚠️ 위험 관리 SA

POSITION RISK_LIMIT P&L
① 상품/한도 마스터 관리 → RISK_LIMIT
④ 주문 체결 처리 → POSITION
⑥ 자산 가격 평가 → P&L
거래·시세 데이터에 의존 (R) → 계산 후 생성

📋 원장 관리 SA

LEDGER TXN_LOG PRICE_HIST
② 시장 시세 수신 → PRICE_HISTORY
④⑤⑦ 다수 기능 → TXN_LOG (모든 이벤트 기록)
⑦ 원장 마감 → LEDGER
모든 SA의 이벤트를 누적 → 이력 관리 중심
✓ 설계 검증: 4개 SA 간 데이터 소유권이 명확히 분리되어 있으며 SA 간 교차 소유는 없다. 참조 의존성(R)은 존재하나 소유권(C)은 단방향으로 유지 — 2단계의 Low Coupling 원칙이 기능-데이터 매핑 레벨에서도 성립한다.

이 단계가 증명하는 것

📌 CRUD 분석이 증명하는 것

"데이터가 어디에 있어야 하는가"를 넘어 "데이터가 어떻게 태어나고, 누가 그 생명에 책임지는가"를 정의하는 과정이다. 생성 소유권의 명확화는 이후 데이터 거버넌스(6단계)의 기술적 근거가 되며, 시스템의 모듈화(Modularity) 수준을 수치로 증명한다.

🎯 Step 4 — 논리 모델로의 연결

CRUD 분석으로 데이터–프로세스 정합성이 검증되었다. 이제 4단계: 논리 모델 & 정규화에서 이 12개 엔터티들 사이의 복잡한 관계 — 슈퍼/서브타입, 재귀 관계, 선분 이력 — 를 정교한 ERD로 표현한다.
